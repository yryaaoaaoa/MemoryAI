# 算法与数据结构（统一参考）

## 一、口述作答标准（文字/语音面试）
1. 先定义问题与状态：输入、输出、约束、目标。
2. 说明选型：为什么用该数据结构（而不是其他结构）。
3. 口述步骤：关键流程、转移关系、剪枝或维护不变式。
4. 复杂度：时间/空间复杂度 + 主瓶颈。
5. 边界与反例：空输入、重复值、极端规模、退化场景。

## 二、数组/链表
- 高频思路：双指针、滑动窗口、前缀和、原地算法。
- 典型追问：如何避免越界；是否能降空间；是否能单次扫描。

## 三、哈希/堆/栈/队列
- 哈希：去重、频次统计、映射索引、冲突与扩容代价。
- 堆：TopK、动态中位数、多路归并，关注 `O(n log k)` 优化。
- 栈：单调栈、括号匹配、表达式求值，不变式要说清。
- 队列：BFS、双端队列、滑窗最值，解释队头有效性。

## 四、树/图
- 树：前中后序、层序、递归/迭代取舍。
- 图：BFS/DFS、最短路、拓扑排序、并查集。
- 典型追问：有环/无环处理、访问标记时机、复杂度上界。

## 五、动态规划（DP）
- 核心：状态定义、转移方程、初始化、遍历顺序。
- 常见失误：状态遗漏、重复计算、滚动数组覆盖顺序错误。
- 追问方向：如何从 `O(n^2)` 优化到 `O(n log n)`（若可行）。

## 六、贪心/回溯
- 贪心：局部最优能否推导全局最优，需给交换论证或反例说明。
- 回溯：搜索树、剪枝条件、去重策略、最坏复杂度评估。

## 七、设计题（算法建模）
- 数据流、在线查询、延迟任务、限流计数等题型。
- 重点：结构选型、正确性、复杂度与工程可落地性。
