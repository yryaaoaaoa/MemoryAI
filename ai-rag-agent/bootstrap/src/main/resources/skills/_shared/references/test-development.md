# 测试开发面试重点（测试基础 + 用例设计 + 自动化 + 性能安全）

## 测试基础
- 软件测试定义与目的：发现缺陷、验证需求、提供质量反馈，不只是"找 Bug"。
- 测试流程：需求分析→测试计划→测试设计→测试执行→缺陷跟踪→测试报告→回归测试。
- 测试左移（Shift-Left）：需求/设计阶段就介入质量活动，早发现早修复，缺陷成本随阶段指数增长。
- 测试覆盖率：行覆盖/分支覆盖/路径覆盖，高覆盖率 ≠ 高质量测试，关注有意义覆盖率而非数字。
- 测试金字塔：单元测试（>90% 自动化）→集成测试（60-80%）→系统/UI 测试（20-40%），投资回报递减。
- 核心竞争力：业务理解 + 系统性用例设计 + 工程化自动化能力。

## 测试用例设计
- 等价类划分：将输入划分为有效/无效等价类，每类取代表性数据，减少冗余。
- 边界值分析：等价类边界的值最易出错，如范围 [1,100] 测试 0/1/100/101。
- 判定表法：多条件组合场景（如会员×节假日×提前天数→价格），穷举条件与结果映射。
- 场景法：模拟用户操作流程（主流程+异常分支），通过流程图覆盖完整业务路径。
- 错误推测法：基于经验推测易错点（如文件上传：空文件/超大文件/病毒文件/格式不符）。
- 状态迁移法：测试有状态变化的对象（如订单状态、电梯状态），验证状态转换正确性。
- 好的测试用例标准：覆盖全面、分组精准（等价类无冗余）、边界清晰。
- 场景题回答框架：功能→兼容性→性能→网络→安全→UI/UX，逐维度展开。

## 测试类型
- 黑盒测试 vs 白盒测试：关注外部行为 vs 关注内部代码逻辑，互补使用。
- 单元测试：最小可测试单元（函数/类），Mock 外部依赖，开发编写，CI 自动执行。
- 集成测试：多模块/服务交互验证，关注接口契约、数据传递、依赖协同。
- 系统测试：完整系统验证，功能/性能/安全/兼容性全面覆盖。
- 回归测试：修改后确认旧功能未被破坏，高度依赖自动化用例。
- 验收测试（UAT）：业务方确认是否满足需求，贴近真实场景。
- 端到端测试（E2E）：完整业务链路验证，从用户入口到最终输出，数量少但价值高。
- 正向测试 vs 反向测试：合法输入验证正常功能 vs 非法输入验证异常处理能力。

## 自动化测试
- ROI 公式：(节省的手工测试时间 × 执行频率) - (脚本开发时间 + 维护成本)。
- 适用场景：回归测试、步骤确定性高、执行频率高；不适用：一次性功能、UI 频繁变动、探索性测试。
- Web 自动化工具：Selenium（多语言/多浏览器、成熟但慢）、Cypress（开发者友好、JS-only）、Playwright（现代化、多语言、快）。
- API 测试流程：读 API 文档→设计用例（正常/异常/边界）→准备数据与认证→发请求→校验响应与数据库。
- API 测试工具：Postman/Reqable/Insomnia，结合 Newman/CI 实现自动化回归。
- 移动端自动化：Appium（跨平台）、Espresso（Android 原生）、XCTest（iOS 原生）。
- CI/CD 集成：Jenkins/GitLab CI/GitHub Actions 中自动触发，测试报告与失败告警。

## 性能测试
- 压测类型：负载测试（预期负载下验证）、压力测试（超负载找极限）、稳定性测试（长时间运行查泄漏）。
- 压测流程：确定目标（QPS/RT/TPS）→准备环境（≈生产 50-80%）→选工具→编写脚本→执行→监控分析→调优→复测。
- 压测工具：JMeter（功能全面）、k6（JS 脚本、现代化）、Gatling（Scala、高性能）、Locust（Python）。
- 关键指标：TPS/QPS、响应时间（P50/P95/P99）、错误率、CPU/内存/磁盘 IO/网络带宽。
- 瓶颈定位：数据库慢查询、连接池打满、线程池饱和、网络带宽、GC 停顿。

## 安全测试
- OWASP Top 10：XSS/SQL 注入/CSRF/身份认证失效/敏感信息泄露/访问控制失效等。
- XSS 测试：输入 `<script>alert(1)</script>`、`<img src=x onerror=alert(1)>`，验证是否转义/过滤。
- SQL 注入测试：`' OR '1'='1 --`、`' UNION SELECT --`、时间盲注 `AND SLEEP(5)`，验证参数化查询。
- CSRF 测试：检查 CSRF Token、SameSite Cookie、Referer 校验。
- 越权测试：水平越权（访问同角色他人数据）、垂直越权（普通用户访问管理接口）。

## Bug 处理与协作
- 不可复现 Bug：多次尝试不同环境/数据→收集详细信息（步骤/日志/截图/录屏）→与开发协作分析。
- 缺陷报告要素：标题、前置条件、复现步骤、预期结果、实际结果、严重程度、优先级。
- Bug 生命周期：新建→确认→分配→修复→验证→关闭（或重新打开）。
- 印象深刻的 Bug 回答模板：场景描述 + 根因分析 + 修复过程 + 后续预防。

## 面试追问模板
- 登录/支付/电商直播间场景，你怎么设计测试用例？
- 你的自动化测试覆盖率是多少？ROI 怎么评估？
- 线上出了一个 Bug，但测试环境无法复现，你怎么排查？
- 你在 CI/CD 中集成了哪些质量门禁？怎么防止质量回退？
