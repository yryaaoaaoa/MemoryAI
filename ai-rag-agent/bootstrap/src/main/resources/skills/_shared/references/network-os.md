# 网络与操作系统（后端面试核心）

## TCP / HTTP 高频点
- TCP 三次握手与四次挥手，TIME_WAIT 的意义与优化边界。
- 粘包/拆包原因与解决（长度字段、分隔符、固定包长）。
- HTTP1.1/2 差异：连接复用、队头阻塞、头部压缩。
- HTTPS 握手流程：证书校验、密钥协商、会话复用。

## 并发与线程模型
- 进程/线程区别，用户态与内核态切换成本。
- 线程池参数设计：核心线程、队列、拒绝策略、监控指标。
- 锁优化：悲观锁/乐观锁、CAS、自旋、锁升级。

## 内存与 I/O
- JVM 内存区域、GC 停顿对接口延迟的影响。
- 零拷贝、mmap、sendfile 的适用场景。
- Reactor 模型与同步阻塞模型差异。

## 面试追问模板
- 为什么不是这个方案？底层代价是什么？
- 如果连接数/流量翻倍，瓶颈会先出现在哪里？
- 出现网络抖动或线程池打满时，如何降级与恢复？
